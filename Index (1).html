// 02 — Developer Portal

Register your bot application

1

Open discord.com/developers/applications

Log in with your Discord account. Click New Application in the top right corner.

2

Add a Bot user

In the left sidebar click Bot → Add Bot → Confirm. This creates the actual bot account.

3

Copy your token

Click Reset Token, copy the result. This is your bot's password — never share it or commit it to Git.

4

Enable Message Content Intent

Scroll down in the Bot section → under Privileged Gateway Intents, enable Message Content Intent.

5

Invite bot to your server

Go to OAuth2 → URL Generator. Check bot and applications.commands. Under Bot Permissions check Administrator. Open the generated URL to invite.

// 04 — Write the Bot

Complete bot.py with events and commands

bot.py
import discord
from discord.ext import commands
from dotenv import load_dotenv
import os
 
load_dotenv()
intents = discord.Intents.default()
intents.message_content = True
intents.members = True
 
bot = commands.Bot(command_prefix="!", intents=intents)
 
# Fires when the bot is connected
@bot.event
async def on_ready():
    print(f"✅ {bot.user} is online in {len(bot.guilds)} servers")
    await bot.change_presence(activity=discord.Game("!help"))
 
# !ping command
@bot.command()
async def ping(ctx):
    await ctx.reply(f"🏓 Pong! {round(bot.latency*1000)}ms")
 
# Welcome new members
@bot.event
async def on_member_join(member):
    ch = member.guild.system_channel
    if ch: await ch.send(f"👋 Welcome {member.mention}!")
 
bot.run(os.getenv("DISCORD_TOKEN"))
✅ Test NowRun python bot.py. See "✅ YourBot is online"? Type !ping in your server to confirm it responds.
// 05 — Slash Commands

Modern slash commands with app_commands

cogs/general.py
import discord
from discord import app_commands
from discord.ext import commands
 
class General(commands.Cog):
    def __init__(self, bot): self.bot = bot
 
    @app_commands.command(name="hello", description="Get a greeting")
    async def hello(self, interaction: discord.Interaction):
        embed = discord.Embed(title="👋 Hey!", color=discord.Color.green())
        await interaction.response.send_message(embed=embed)
 
async def setup(bot): await bot.add_cog(General(bot))

🔄 Sync your commands

Add await bot.tree.sync() inside on_ready. For instant dev testing, sync to a specific guild: await bot.tree.sync(guild=discord.Object(id=GUILD_ID)) — guild-synced commands appear instantly. Global sync can take up to an hour.

// 03 — Core Bot

Bot + GPT-4o integration

bot.py
from telegram.ext import ApplicationBuilder, CommandHandler, MessageHandler, filters, ContextTypes
from openai import AsyncOpenAI
from telegram import Update
import os
 
client = AsyncOpenAI(api_key=os.getenv("OPENAI_API_KEY"))
history = {} # per-user conversation memory
 
async def reply(update: Update, ctx: ContextTypes.DEFAULT_TYPE):
    uid = update.effective_user.id
    if uid not in history:
        history[uid] = [{"role":"system","content":"You are a helpful Telegram bot."}]
    history[uid].append({"role":"user","content":update.message.text})
    res = await client.chat.completions.create(model="gpt-4o", messages=history[uid], max_tokens=500)
    text = res.choices[0].message.content
    history[uid].append({"role":"assistant","content":text})
    await update.message.reply_text(text)
 
app = ApplicationBuilder().token(os.getenv("TELEGRAM_TOKEN")).build()
app.add_handler(MessageHandler(filters.TEXT, reply))
app.run_polling()
✅ It has memory!The history dict stores each user's full conversation so GPT-4o remembers context across multiple messages — just like ChatGPT. Cap at last 20 messages to control costs.

// 02 — Basic Scraper

Scrape any page, extract data

scraper.py
from playwright.async_api import async_playwright
from bs4 import BeautifulSoup
import pandas as pd, asyncio
 
async def scrape(url: str) -> list:
    async with async_playwright() as p:
        browser = await p.chromium.launch()
        page = await browser.new_page()
        await page.goto(url)
        # Wait for JS to render (key for SPAs)
        await page.wait_for_load_state("networkidle")
        html = await page.content()
        await browser.close()
 
    soup = BeautifulSoup(html, "html.parser")
    items = []
    for card in soup.select(".product-card"): # change selector
        items.append({
            "name": card.select_one("h2").text.strip(),
            "price": card.select_one(".price").text.strip()
        })
    return items
 
# Run and save to CSV
data = asyncio.run(scrape("https://example.com"))
pd.DataFrame(data).to_csv("output.csv", index=False)
// 03 — Schedule It

Run automatically on a timer

schedule_runner.py
import schedule, time, asyncio
 
def job():
    data = asyncio.run(scrape("https://example.com"))
    save_to_db(data) # your storage function
 
schedule.every(6).hours.do(job)
while True:
    schedule.run_pending()
    time.sleep(60)
⚠️ Be respectfulAlways check a site's robots.txt before scraping. Add delays between requests (await asyncio.sleep(2)) and set a realistic User-Agent header. Never scrape login-protected data without permission.

// 01 — What is an Agent?

Agents vs. basic chatbots

A regular chatbot just replies to messages. An agent decides which actions to take, executes tools (like searching the web, running Python code, reading files), observes the results, and continues reasoning until it reaches a final answer. This is the ReAct pattern — Reason + Act.

🔄 The ReAct Loop

Thought: "I need to find today's BTC price."
Action: Call web_search("bitcoin price today")
Observation: "$67,420"
Thought: "I have the answer now."
Final Answer: "Bitcoin is currently at $67,420."

// 02 — Build It

LangChain agent with web search + code execution

bash — install
pip install langchain langchain-openai duckduckgo-search
agent.py
from langchain_openai import ChatOpenAI
from langchain.agents import AgentExecutor, create_react_agent
from langchain_community.tools import DuckDuckGoSearchRun
from langchain.tools import tool
from langchain import hub
 
llm = ChatOpenAI(model="gpt-4o", temperature=0)
 
# Tool 1: web search
search = DuckDuckGoSearchRun()
 
# Tool 2: Python calculator
@tool
def calculate(expression: str) -> str:
    """Evaluate a Python math expression."""
    return str(eval(expression))
 
tools = [search, calculate]
prompt = hub.pull("hwchase17/react")
agent = create_react_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True)
 
result = executor.invoke({"input": "What is Bitcoin's price? Calculate 10% of it."})
print(result["output"])
✅ It reasons step by stepSet verbose=True to watch the agent's thought process in your terminal. You'll see every Thought → Action → Observation cycle printed live.

// 01 — Setup

Twitter Developer App + Tweepy

1

Apply for a Developer Account

Go to developer.twitter.com. Apply for Basic access (~$100/mo) or use the free tier which allows posting only. Create a Project and App.

2

Get your 4 keys

From your App's dashboard, copy: API_KEY, API_SECRET, ACCESS_TOKEN, ACCESS_TOKEN_SECRET. Store all in .env.

twitter_bot.py
import tweepy, os, schedule, time
from dotenv import load_dotenv
load_dotenv()
 
client = tweepy.Client(
    consumer_key=os.getenv("API_KEY"),
    consumer_secret=os.getenv("API_SECRET"),
    access_token=os.getenv("ACCESS_TOKEN"),
    access_token_secret=os.getenv("ACCESS_SECRET")
)
 
# Post a tweet
def post_daily_tip():
    tips = ["Tip 1: Always handle errors!", "Tip 2: Write tests!"]
    client.create_tweet(text=tips[0]) # rotate from list
 
schedule.every().day.at("09:00").do(post_daily_tip)
while True: schedule.run_pending(); time.sleep(30)
⚠️ Rate limitsTwitter's free API allows only 500 tweets/month. Basic tier ($100/mo) allows 3,000/month. Never spam — accounts get suspended quickly. Use scheduling to space posts out.

Embeddable Website Chatbot Widget
FastAPI · OpenAI · Vanilla JS · One-line embed · Intermediate · ~2.5 hrs
Web App ▾
// 01 — FastAPI Backend

Build the API server

main.py — FastAPI server
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from openai import AsyncOpenAI
from pydantic import BaseModel
 
app = FastAPI()
app.add_middleware(CORSMiddleware, allow_origins=["*"], allow_methods=["*"])
client = AsyncOpenAI()
 
class ChatMsg(BaseModel):
    message: str
    history: list = []
 
@app.post("/chat")
async def chat(body: ChatMsg):
    msgs = [{"role":"system","content":"You are a helpful website assistant."}]
    msgs += body.history
    msgs.append({"role":"user","content":body.message})
    res = await client.chat.completions.create(model="gpt-4o-mini", messages=msgs)
    return {"reply": res.choices[0].message.content}
// 02 — Embed Widget

One-line embed for any website

widget.js — self-contained embed
/* Drop this one line anywhere in your site's HTML: */
<script src="https://your-api.com/widget.js"></script>
 
/* widget.js creates a floating chat button + panel */
(function() {
  const btn = document.createElement('button');
  btn.innerHTML = '💬';
  btn.style.cssText = 'position:fixed;bottom:24px;right:24px;width:56px;height:56px;border-radius:50%;background:#4ade80;border:none;font-size:1.4rem;cursor:pointer;z-index:9999;box-shadow:0 4px 20px rgba(0,0,0,0.2)';
  document.body.appendChild(btn);
  // ... chat panel and API calls follow
})()
✅ Works on any siteThe widget is self-contained — it creates its own DOM elements, styles, and handles all API communication. No framework needed on the host site.
